v2.19.4
Calendar: stop the countdown tick clobbering the real error. The "Calendar not configured" message appeared even with a valid URL because the 1-second countdown tick re-rendered with no error context and hit a catch-all message. Calendar: fix "not configured" despite a saved URL. v2.19.2
Calendar: render robustness + field styling. The iCal URL field now matches the other settings inputs. The card shows whenever a URL is set (removed the separate enable toggle that could leave it blank). Broadened Google host permissions to cover iCal redirects (likely cause of live fetch failing). Fetch errors now show a specific reason instead of a blank card.

Phase 2 (meeting hours alongside logged hours) is deferred; note that Zeal leave lives in Leapsome so this shows meetings, not approved absence.
v2.18.0
Usage analytics enrichment (T-UA-1): rolling per-user profile + action tracking. Each app_opened now carries a self-describing profile - days active, lifetime opens, first/current version, and per-section/action counts - kept in local storage and attached to the event, so Sentry's plain Issues view answers "since when, how many days, how often, what version" per user without Discover-tier aggregation. Three meaningful actions are now tracked (export_report, scope_toggled, ticket_clicked) as action_taken events with an action tag. Demo mode stays excluded. section and role tags were verified healthy - no fix needed.
