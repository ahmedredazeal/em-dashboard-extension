EM Dashboard — Changelog

v1.3.6
Chart polish + refresh loading state: Full display names in timesheet chart (Abdelrahman Rezk instead of Abdelrahm). Smaller chart dimensions. Refresh button now shows loading state on sprint, Sentry, and extra board sections simultaneously. Worklog fetch wrapped with 15s timeout to prevent service worker crashes.
v1.3.5
Timesheet: worklogDate JQL covers subtasks in one call: Team members log time on subtasks (e.g. [FE] Implementation) which are not in the sprint directly. New getSprintWorklogs() uses JQL worklogDate >= start AND <= end to get ALL worklogged issues including subtasks with a single API call.
v1.3.4
Timesheet: fetch all sprint worklogs per-issue + doc fixes: Worklog inline field only returns data for issues where someone explicitly logged time. Now fetches worklogs individually for every sprint issue in parallel, giving complete team coverage. Manifest version corrected; changelog.html entries for v1.3.1–v1.3.3 added.
v1.3.3
Fix: dynamic import() disallowed in service worker: await import() inside a service worker is banned by the HTML spec. Fixed by adding dayIndex to the static import at the top of background.js.
v1.3.2
Analytics fixes: Burndown actual line fallback using ticket updated date for done stories without changelog close timestamp. Timesheet redesigned as horizontal bar chart (scales to any number of members). Sprint Analytics section moved above ticket list and collapsed by default.
v1.3.1
Fix: expand=changelog as URL query param: Jira POST /rest/api/3/search/jql ignores expand in the request body and returns 400. Fixed by appending ?expand=changelog to the URL instead.
v1.3.0
Sprint Analytics Charts: New collapsible section under Current Sprint (expanded by default) showing a burndown chart with 3 series (ideal/by-due-date/actual from changelog) and a timesheet grouped bar chart (week1 vs week2 hours per member from Jira worklogs). Sprint-change banner prompts to keep or delete old analytics. Data cached per sprint name.
v1.2.9
Sprint name in title + ticket counts on headers: Current Sprint section title shows sprint name in brackets — Current Sprint (HRM Sprint 64) — with total ticket count on the right. Extra board titles show 100 TOTAL on the right. Consistent with Sentry's section header style.
v1.2.8
Real status counts + Kanban board filter fix: Status summary now shows actual status names from your data (QA Accepted, QA Rejected, In Progress, QA Testing, Open…) instead of trying to bucket by statusCategory. Support/Kanban extra boards now fetch via the board's own filter JQL — fixes empty board data in Jira Cloud.
v1.2.7
Ticket counts in collapsed header + statusCategory fix: Fixed critical bug — Jira statusCategory for in-progress is indeterminate not inprogress, causing "0 In Progress" even when tickets were in progress. Renamed "closed" → "QA Accepted". Ticket counts (X QA Accepted · X In Progress · X Open) and support badges now appear in the collapsed header — always visible without expanding.
v1.2.6
Unified sections + priority + clickable tickets: Priority colour dot on every ticket (red→orange→amber→blue→grey). All Jira ticket rows are clickable (opens Jira in browser). Ticket summary shows X closed / in progress / open at top of each board. Support boards show BreachedSLA 🔴 and blocked-external ⚠ label counts. "Current Sprint" and extra boards now use the same section-label + collapsible style as Sentry.
v1.2.5
Incremental rendering: Sprint and Sentry sections now update independently. Jira data (sprint, extra boards) renders as soon as it arrives. Sentry renders separately when its slower fetch completes. No more waiting for both sources before anything shows.
v1.2.4
Actual fix for extra boards: Root cause was the 2-minute cache grace window — saving settings triggered a popup reload but the fresh-fetch was skipped because the cache was less than 2 minutes old, so old data (with no extra boards) was rendered. Fix: settings-updated handler now zeros the cache timestamp before reloading, forcing a fresh fetch every time. Added 12 integration tests covering the full settings→cache→fetch→render flow.
v1.2.3
Unit tests + verbose logging: Extracted parsing logic into pure functions in src/parsers.js with 32 unit tests covering all input shapes (object, string, null, invalid). Added tests/parsers.test.js and wired it into pre-flight — no release ships if a test fails. Verbose logging added to the extra-boards fetch pipeline so the service worker console reveals exactly where data flow breaks.
v1.2.2
Fix: extra boards not rendering (two scope bugs): (1) storyPointsField was declared const inside a try{} block — block-scoped, so the extra boards loop couldn't see it. Fixed by declaring with let before the try block. (2) Loop variable boardId silently overwrote the outer boardId (main sprint board). Renamed to extraBoardId. Both bugs caused extra boards to fail silently with no render.
v1.2.1
Fix: ReferenceError state is not defined: fetchJiraData() was referencing state.extraBoardsData but state only exists inside checkDashboard(). This caused every Jira fetch to throw a ReferenceError, meaning sprint data never rendered. Fixed by using a local extraBoardsData array inside fetchJiraData() and returning it as part of the result.
v1.2.0
Critical syntax fix + pre-flight brace check: Fixed missing closing } in fetchJiraData that caused "Service worker registration failed / Unexpected end of input" on load. Added brace balance check to pre-flight script — node --check silently misses this class of error in ES modules, so now both checks run on every file before packaging.
v1.1.9
Extra boards fully working: Fixed broken pipeline (data was fetched but never saved to storage or rendered). Extra boards now appear as collapsible sections below the main sprint — showing board label, sprint name, point progress, and full story list with assignee, points, and due dates. Settings now accept Name|BoardID format (one per line).
v1.1.8
Due dates + icon fix + decimal fix: Sprint stories now show due dates inline (📅 15 May), with red highlighting for overdue tickets and amber for due within 2 days. Expected velocity no longer shows floating-point noise (4.1pt/day, not 4.076923...). Tab icon regenerated with correct RGBA format.
v1.1.7
Refresh timer fixed: Fixed visibility bug (timer now always shows in context bar). Under 5 min since last refresh shows "just now" or "Xm ago". At 5+ min switches to a live mm:ss countdown to the 30-min mark (goes amber under 5 min remaining). Countdown hitting zero automatically triggers a refresh.
v1.1.6
On-demand fetching — no background alarm: Removed the 30-minute background alarm. Data now fetches only when you open the panel (skips if cache is under 2 minutes old). The elapsed timer beside ↻ now shows "just now / Xm ago / Xh ago" instead of a countdown. No more unnecessary API calls while the panel is closed.
v1.1.5
Refresh countdown timer: Small mm:ss countdown appears beside the ↻ button in the context bar, showing time until next auto-refresh. Reads last fetch timestamp from cache for accuracy. Resets on manual refresh. Data refreshes every 30 minutes automatically.
v1.1.4
Collaboration Infrastructure + GitHub Pages: Added CONTRIBUTING.md (workflow + versioning rules), HANDOFF.md (session state log for contributors), AI_PROMPT.md (onboarding prompt for Claude sessions), GitHub Issue templates, and a GitHub Pages landing page (index.html) hosting docs and changelog. Versioning is now mandatory on every push — PATCH for fixes, MINOR for features, MAJOR for breaking changes.
v1.1.3
Per-View Sentry Sections + Auto Story Points: Sentry issues now render as separate collapsible sections per view — click a view header to expand/collapse. Story points auto-detected from board configuration (no manual field ID needed). Loading indicators shown while data fetches. Version number synchronized with manifest.json. Issues show project slug and assignee.
v1.1.0
Multi-Project Sentry + Auto-Reload: Settings now support comma-separated Sentry saved view IDs for issues across multiple projects. Side panel auto-reloads after settings save (no manual close/reopen needed). Added detailed logging for active sprint debugging. Migration utility for future data model upgrades. Foundation work for dynamic board configuration (full UI coming in v1.2.0).
v1.0.0
Phase 1 — Foundation: Jira + Sentry integration, side panel UI with Today/Sprint Health/Reliability screens, 4 core alert rules (velocity drop, sprint goal at risk, Sentry spike untriaged, support SLA breach), toolbar badge, privacy mode toggle for screen sharing, two-row sticky header system, light/dark/browser themes. Built with zero infrastructure — credentials stored locally, read-only APIs, no backend.
